"{~text align=center}{Справка по Банкам существ}}:\nЭтот плагин содержит модуль для расширенной поддержки Банков Существ, который позволяет изменять текущие и добавлять новые Банки существ в Heroes III ERA, работая как с оригинальными наградами, так и привнося совершенно новые, кастомные награды. Теперь помимо стандартных артефактов, ресурсов и существ, можно задать дополнительное вознаграждение, включая опыт, очки заклинаний, удачу, мораль и даже заклинания.\nНиже предоставлены все возможные ключи, которые влияют на реализованные параметры Банки Существ.\nВ данном примере будет использоваться объект карты приключений с подтипом {~r}22} и добавлением всевозможных наград:\n\n\"{name}\" - строка: устанавливает имя Банку Существ. Пример использования:\n\"{RMG.objectGeneration.16.22.name}\": \"Мифический Храм\"\n\nДалее необходимо установить 4 состояния Банка, одно из которых (0-3) случайно выбирается для каждого из них на старте карты (или с помощью иных скриптов) в ф-ции \"{void __fastcall H3CreatureBank::SetByType(H3CreatureBank *bank, int cbType)} по адресу {0x0047A6C0}\":\nВ примере будет использоваться 'состояние' с индексом {~r}1}:\n\n\"{artifactTypeCounts}\" - массив из 4 чисел: отвечает за количество случайных артефактов разной ценности (ценный, малый, великий и реликтовый). Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.artifactTypeCounts}\": [1,1,3,0]\n\n\"{resources}\" - массив из 8 чисел: количество получаемых ресурсов (древесина, руда, ртуть, сера, кристаллы, самоцветы, золото, мифрил). Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.resources}\": [5,5,2,2,1,1,10000,50]\n\n\"{creatureRewardType}\" - число: ID существа, получаемого в награду. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.creatureRewardType}\": 25\n\n\"{creatureRewardCount}\" - число: количество существ в награде. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.creatureRewardCount}\": 12\n\n\"{guardians.type}\" - массив из 7 чисел: список существ, охраняющих банк. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.guardians.type}\": [55, 55, -1, -1, -1, -1, -1]\n\n\"{guardians.count}\" - массив из 7 чисел: количество существ-охранников. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.guardians.count}\": [25, 20, 0, 0, 0, 0, 0]\n\n\"{upgrade}\" - число: шанс на улучшенную версию существ (0-100). Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.upgrade}\": 50\n\n\"{chance}\" - число: вероятность выбора данного состояния банка. Сумма всех состояний должна быть 100. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.chance}\": 40\n\n\"{skills.primary}\" - массив из 4 чисел: число добавляемых первичных навыков. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.skills.primary}\": [1,0,1,3]\n\n\"{experience}\" - число: опыт, получаемый после разграбления. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.experience}\": 5000\n\n\"{spellPoints}\" - число: количество добавляемых очков заклинаний. Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.spellPoints}\": 20\n\n\"{luck}\" - число: изменение удачи (от -3 до 3). Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.luck}\": 1\n\n\"{morale}\" - число: изменение морали (от -3 до 3). Пример использования:\n\"{RMG.objectGeneration.16.22.states.1.morale}\": -1\n\n\"{spells}\" - массив из 4 объектов заклинаний: можно использовать либо точный индекс заклинаний в награду, либо генерировать случайное заклинание по битовому сету школы магии [1-15], уровню заклинания [1-31], флагам заклинания [1-4294967295]. Если ключ пустой, то ограничения игнорируются. Пример использования точного id:\n\"{RMG.objectGeneration.16.22.states.1.spells.0.id}\": 15,\n\"Пример использования битовых наборов для генерации заклинаний только школы магии воды и лишь 2 или 3 уровня (2*2 +2*3 = 12):\n {RMG.objectGeneration.16.22.states.1.spells.0.bits.schools}\": 4,\n\"{RMG.objectGeneration.16.22.states.1.spells.0.bits.levels}\": 12"
